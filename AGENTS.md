# AGENTS.md

本仓库用于实现一个“网页版拓麻歌子 / 线上电子宠物平台”MVP。
你是本项目的全栈开发代理，目标是：先做出可运行、可测试、可部署的最小闭环，再逐步扩展玩法。

## 项目目标
实现一个支持多用户注册登录、创建宠物、照顾宠物、开启托管、加入小圈子并发生异步互动的网站。
产品保留拓麻歌子核心：养成、成长、关系、代际、轻社交、异步事件。

## 技术栈硬约束
- 后端：Python 3.11 + FastAPI
- ORM：SQLAlchemy
- 数据库：SQLite（MVP）
- 前端：Jinja2 + HTMX + 简洁 CSS
- 测试：pytest + Playwright
- 部署：Debian + python venv + systemd
- 默认服务监听：0.0.0.0:18427
- 不要引入 Docker、Redis、Celery、PostgreSQL，除非我明确要求
- 所有页面文案、README、注释尽量使用中文

## 开发优先级
1. 先让网站能跑
2. 先做单用户完整闭环，再做多人互动
3. 先做异步社交，不做实时聊天
4. 先做可解释的托管，再做复杂 AI 行为
5. 先做结构清晰的代码，再做 UI 美化

## 功能范围
### 必须做
- 注册 / 登录 / 退出
- 用户创建宠物
- 宠物状态：hunger, mood, cleanliness, health, energy
- 宠物生命周期：egg, baby, child, teen, adult
- 基础操作：喂食、玩耍、清理、睡觉、治病
- 时间推进：根据 last_tick_at 自动补算状态
- 托管模式：保命托管、生活托管、社交托管
- 小圈子：创建、加入、成员列表、异步互动
- 互动动作：拜访、送礼、一起玩、结伴任务
- 简化代际：成年且关系足够可繁殖下一代
- 商店：固定商品
- 事件日志
- /health 健康检查
- 自动化测试：至少覆盖注册、登录、创建宠物、喂食、开启托管、加入圈子、互动流程

### 暂不做
- 自由聊天
- 用户上传图片
- 开放交易市场
- 复杂基因算法
- WebSocket 实时系统
- 管理后台复杂权限系统

## 代码规范
- 路由层只处理请求/响应，不写重业务逻辑
- 业务逻辑放到 services
- 数据校验放到 schemas
- 所有关键操作写 pet_events
- 所有状态推进统一走 tick / simulation service
- 命名统一、可读，不要缩写成难懂变量
- 不制造死代码和未使用文件
- 不为了“显得高级”引入复杂依赖

## 状态推进规则
默认按小时补算：
- hunger 随时间上升
- mood 随饥饿和脏乱下降
- cleanliness 随时间下降
- energy 依据清醒/睡眠状态变化
- health 受到 hunger、cleanliness、energy 联动影响
- 状态值统一用 0~100
- 所有补算必须是幂等的：同一时间窗口重复调用结果一致

## 托管规则
### 保命托管
- 仅在健康风险较高时做最低限度干预
- 优先避免死亡或严重疾病

### 生活托管
- 自动维持较平稳状态
- 按预设阈值自动喂食、清洁、睡觉、用药

### 社交托管
- 继承生活托管
- 在圈子内自动参与结构化互动
- 必须有冷却时间，避免刷分

## 社交规则
- 圈子是轻社交单位
- 互动必须异步、结构化
- 不允许自由文本聊天
- 每次互动应写事件日志
- 互动行为应影响 relationship_score、mood、coins 或奖励

## 代际规则
- 只有 adult 才可繁殖
- 需要关系值达到阈值
- 下一代继承部分颜色倾向、性格倾向和少量属性加成
- 必须能查看简单家谱

## 安全要求
- 使用密码哈希
- 敏感配置放入 .env
- 表单输入做基本校验
- 不把密钥写死
- 关键写操作做基础 CSRF 或等价保护
- session cookie 需合理配置

## 测试要求
必须提供：
- pytest 单测或集成测试
- Playwright E2E 测试
- seed 脚本：创建测试用户、宠物、圈子、商品
- 一键测试脚本
- README 中给出运行测试的命令
