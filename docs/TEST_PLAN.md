# 自动化测试方案

本文档定义 Tamapet 项目的测试目标、测试分层、执行命令、验收标准与排错方式。  
目标不是只验证接口返回 200，而是验证“这个宠物网站是否真的能被用户正常使用”。

---

## 1. 测试目标

本项目测试要覆盖以下核心闭环：

1. 首页可访问
2. 用户可以注册
3. 用户可以登录
4. 用户可以创建宠物
5. 用户可以对宠物执行基础照顾动作
6. 用户可以开启托管
7. 用户可以创建或加入圈子
8. 用户可以触发至少一次圈子互动
9. 健康检查接口正常
10. 网站在 Debian 上可启动、可访问、可重复验证

---

## 2. 测试分层

本项目测试分为 3 层：

### 2.1 健康检查 / 基础接口测试
目标：
- 确认服务已经启动
- 核心接口未直接报错
- 适合作为最基础的 smoke test

覆盖内容：
- GET /health
- 首页 GET /
- 登录页 GET /login
- 注册页 GET /register

### 2.2 后端逻辑测试
目标：
- 验证核心业务规则
- 防止状态推进、托管逻辑、互动冷却逻辑出错

建议覆盖：
- 宠物状态推进 service
- 宠物成长阶段变更
- 托管模式计算逻辑
- 圈子互动冷却与奖励逻辑
- 繁殖/代际条件判断

### 2.3 端到端 E2E 测试
目标：
- 从真实用户视角验证整站流程
- 用浏览器自动完成点击、输入、跳转、提交和断言

使用工具：
- pytest
- Playwright
- Chromium

---

## 3. 当前必须通过的 E2E 流程

以下流程为 MVP 阶段的最低自动化验收标准：

### 流程 A：首页可访问
- 打开首页
- 确认页面成功加载
- 确认页面主体内容可见

### 流程 B：注册 + 登录
- 进入注册页
- 输入邮箱、密码、确认密码
- 注册成功
- 跳转或进入登录页
- 使用新账号登录成功
- 进入 dashboard

### 流程 C：创建宠物
- 从 dashboard 进入新建宠物页
- 输入宠物名字
- 选择物种、颜色、性格
- 提交创建
- 页面出现宠物信息

### 流程 D：照顾宠物
- 点击“喂食”
- 点击“玩耍”
- 点击“清理”或“睡觉”
- 页面提示动作成功
- 宠物状态或事件日志发生变化

### 流程 E：托管
- 进入托管设置页
- 选择一种托管模式
- 提交保存
- 页面提示“托管设置已保存”

### 流程 F：圈子
- 进入圈子页面
- 创建圈子或加入默认圈子
- 成功进入圈子详情页
- 触发一次互动动作
- 页面出现互动结果或事件日志

### 流程 G：健康检查
- 请求 `/health`
- 返回 JSON
- 状态为正常
