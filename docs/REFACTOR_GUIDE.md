# 方块 FPS 重构指南

本文档用于指导 `app/static/voxel-fps.js` 的后续拆分，不涉及大规模重写。

## 当前问题

- 单文件状态过多
- 输入、房间、HUD、结算逻辑耦合在一起
- 触屏与桌面端交互共享状态较多

## 推荐拆分顺序

### 1. 输入层
优先拆出：
- 桌面键鼠输入
- 移动端摇杆与按钮输入
- Pointer Lock / 全屏控制

### 2. 联机层
优先拆出：
- 房间 API 调用
- WebSocket 连接与消息处理
- 房间状态缓存与恢复

### 3. 结算层
优先拆出：
- MVP 计算
- 战斗结果展示
- 单机 / 联机结算差异

### 4. UI 层
优先拆出：
- Overlay 流程
- HUD 更新
- 移动端按钮状态

## 原则

- 一次只拆一个边界
- 先补回归测试再拆
- 不在没有验证的情况下重写核心循环
